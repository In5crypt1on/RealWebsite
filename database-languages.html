Database Languages

Languages for querying and managing data.

Database languages, or query languages, are used to retrieve, manage, and manipulate data stored in a database. The most common type is SQL, used for relational databases (like tables in a spreadsheet). Modern NoSQL databases often use their own JSON-like query languages for more flexible data structures.

SQL

Structured Query Language is the universal standard for managing and querying data in relational databases (data organized in tables).

  • Declarative: You describe *what* data you want, and the database engine figures out *how* to get it.
  • Relational Databases: Used by systems like MySQL, PostgreSQL, SQL Server, and SQLite to manage structured data with predefined schemas.
  • Powerful Joins: Its most powerful feature is the ability to combine rows from two or more tables based on a related column, enabling complex queries.
-- Selects all users from the 'users' table
-- who are older than 30.
SELECT * FROM users WHERE age > 30;

MongoDB QL

MongoDB uses a rich, JSON-based query language to interact with its document-oriented database, where data is stored in flexible, JSON-like documents.

  • Document-Based: Queries operate on "documents" which map naturally to objects in most programming languages.
  • Horizontal Scaling: NoSQL databases like MongoDB are generally easier to scale across multiple servers to handle massive amounts of data.
  • Use Cases: Ideal for applications with rapidly changing requirements, large amounts of unstructured data, and real-time applications.
// Finds all documents in the 'users' collection
// where the age is greater than 30.
db.users.find({ age: { $gt: 30 } })
